## Base URL

All API requests are made to the following base URL:

```
https://vinecms.tech/api/public/v1
```

## API Versioning

The Vine Public Content API is versioned. `v1` is the current stable version. The version should be included in the URL path immediately after `/api/public`.

**Supported URL formats:**
- **Current**: `https://vinecms.tech/api/public/v1/{API_KEY}/posts`

Use the explicit `/v1/` format.

## Authentication

The Vine Public Content API uses API key authentication. Your API key is included directly in the URL path.

### Getting Your API Key

You can obtain your API key from your Vine workspace. The key is read-only and maps directly to your workspace, so keep it private.

### Using Your API Key

Replace `{API_KEY}` in each endpoint URL with your actual API key. For example:

```bash
# Before
https://vinecms.tech/api/public/v1/{API_KEY}/posts

# After (with your actual key)
https://vinecms.tech/api/public/v1/your-actual-key-here/posts
```

## Making Requests

The public API is configured for simple GET requests. No special headers are required, so you can call it directly from:

- **Browser**: Use `fetch()` or `XMLHttpRequest`
- **cURL**: Use the command line examples provided in each endpoint
- **Any HTTP client**: Standard GET requests work out of the box

### Example Request

Here's a simple example using cURL:

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/posts"
```

Or using JavaScript in the browser:

```js
const response = await fetch(
  'https://vinecms.tech/api/public/v1/{API_KEY}/posts'
);
const data = await response.json();
console.log(data);
```

## Response Format

All successful responses return JSON data. Error responses also use JSON.

### Success Response

```json
{
  "ok": true,
  "workspace": {
    "id": "workspace_id",
    "name": "Workspace Name",
    "slug": "workspace-slug"
  },
  "posts": []
}
```

### Error Response

```json
{
  "ok": false,
  "error": "Invalid API key"
}
```
