The current public API exposes two post endpoints.

## List Posts

### Endpoint

```text
GET https://vinecms.tech/api/public/v1/{API_KEY}/posts
```

### Example

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/posts"
```

### Response

```json
{
  "ok": true,
  "workspace": {
    "id": "workspace_id",
    "name": "Vyeos",
    "slug": "vyeos"
  },
  "posts": [
    {
      "id": "post_id",
      "title": "Welcome to Vine",
      "slug": "welcome-to-vine",
      "excerpt": "A short introduction to the workspace.",
      "readingTimeMinutes": 2,
      "publishedAt": "2026-03-08T10:00:00.000Z",
      "updatedAt": "2026-03-08T10:00:00.000Z",
      "author": { "id": "author_id", "name": "Ava Stone" },
      "category": { "slug": "announcements", "name": "Announcements" },
      "tags": [{ "slug": "welcome", "name": "Welcome" }]
    }
  ]
}
```

## Get Post By Slug

### Endpoint

```text
GET https://vinecms.tech/api/public/v1/{API_KEY}/posts/{postSlug}
```

### Path Parameters

| Parameter | Required | Description |
|-----------|----------|-------------|
| `postSlug` | Yes | The post slug |

### Example

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/posts/welcome-to-vine"
```

### Response

```json
{
  "ok": true,
  "workspace": {
    "id": "workspace_id",
    "name": "Vyeos",
    "slug": "vyeos"
  },
  "post": {
    "id": "post_id",
    "title": "Welcome to Vine",
    "slug": "welcome-to-vine",
    "excerpt": "A short introduction to the workspace.",
    "readingTimeMinutes": 2,
    "publishedAt": "2026-03-08T10:00:00.000Z",
    "updatedAt": "2026-03-08T10:00:00.000Z",
    "author": { "id": "author_id", "name": "Ava Stone" },
    "category": { "slug": "announcements", "name": "Announcements" },
    "tags": [{ "slug": "welcome", "name": "Welcome" }],
    "contentHtml": "<p>Hello world</p>",
    "contentJson": null
  }
}
```

## Reading Time

Every post summary and full post now includes `readingTimeMinutes`.

The estimate is based on:

- text at roughly `225` words per minute
- extra time for images
- extra time for code blocks

So it is more realistic than a simple `wordCount / 225` estimate, especially for rich posts.

## Visibility Rules

These endpoints only return posts that are:

- `status = "published"`
- `visible = true`

Drafts, hidden posts, and missing slugs are not returned.

## Not Implemented Yet

The current posts API does not yet support:

- pagination
- tag filtering
- category filtering
- author filtering
- search
