## 401 Invalid API Key

Current error:

```json
{
  "ok": false,
  "error": "Invalid API key"
}
```

Check:

1. The API key is in the path.
2. The API key belongs to an existing workspace.

Correct example:

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/posts"
```

Wrong examples:

```bash
curl "https://vinecms.tech/api/public/v1/posts"
curl "https://vinecms.tech/api/public/v1/{API_KEY_MISSING_RESOURCE}"
```

## 404 Post Not Found

Current error:

```json
{
  "ok": false,
  "error": "Post not found"
}
```

This can mean:

- the slug is wrong
- the post is not `published`
- the post is not `visible`

Try listing posts first:

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/posts"
```

## Empty Post List

If you get:

```json
{
  "ok": true,
  "workspace": {
    "id": "workspace_id",
    "name": "Vyeos",
    "slug": "vyeos"
  },
  "posts": []
}
```

check whether the workspace has posts that are both:

- `published`
- `visible`

Drafts and hidden posts are excluded.

## Route Shape

The current route shape is:

```text
/api/public/v1/{API_KEY}/posts
```

If you are calling older shapes, update them.

## Other Endpoints

These endpoints are now available:

- `/authors`
- `/categories`
- `/tags`
- `/stats`

Example checks:

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/authors"
curl "https://vinecms.tech/api/public/v1/{API_KEY}/categories"
curl "https://vinecms.tech/api/public/v1/{API_KEY}/tags"
curl "https://vinecms.tech/api/public/v1/{API_KEY}/stats"
```
